# scripts/

Helper scripts for `game-server-deploy` maintainers. Part of the repo's npm
workspace ‚Äî installed with a single `npm install` from the repo root.

## User-facing: `init-parent.mjs` release asset

The **pre-built release asset** is the recommended entry point for end users.
It requires only Node.js 20+ and no npm install:

```bash
# Bootstrap a fresh parent repo (git init + submodule add + scaffold)
curl -fsSL https://github.com/CoderCoco/game-server-deploy/releases/latest/download/init-parent.mjs \
  -o init-parent.mjs
node init-parent.mjs bootstrap

# Migrate existing wrapper files to the latest templates
node init-parent.mjs --migrate
# or equivalently:
node init-parent.mjs migrate

# Show help
node init-parent.mjs --help
```

The `Makefile` generated by `init-parent` automatically fetches the latest
`init-parent.mjs` and runs `--migrate` as part of `make update`.

## Subcommands

| Subcommand   | Description |
|--------------|-------------|
| `init`       | Interactive scaffolding for a new parent-repo (default when no subcommand given) |
| `bootstrap`  | git init + optional `gh repo create` + `git submodule add` + `init` |
| `migrate`    | Re-render `Makefile` and `.gitignore` against the latest templates (reads existing values) |

### Build the bundle

```bash
npm run scripts:build          # from repo root
# or
npm run build                  # from scripts/
```

Output: `scripts/dist/init-parent.mjs` (gitignored ‚Äî release artifact).

### Run tests

```bash
npm run scripts:test           # from repo root
```

### Requirements

- Node.js 20+
- `git` on `$PATH`

Windows users should run under WSL or Git Bash ‚Äî the generated `Makefile`
uses `bash`, `sha256sum`, and `cp`.
